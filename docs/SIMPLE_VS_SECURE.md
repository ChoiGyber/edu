# Root vs 사용자 계정: 당신의 상황 분석

## 당신의 상황
- 개인 프로젝트
- 혼자 운영
- 처음부터 구성

**결론: 간단하게 하는 것이 훨씬 낫습니다.**

---

## 상세 비교

### 1) Root 계정으로 운영

**설정 시간:** 5~6분

**장점**
- 설정이 매우 간단함
- 모든 명령어 바로 실행 가능
- 파일 권한 문제 없음
- 배포 빠름
- 초기 학습 곡선 낮음

**단점**
- 키 유출 시 전체 서버 장악 가능
- 실수 시 영향 큼
- 감시/감사 안 됨

**실제 위험도: 낮음**
- 개인 서버
- 혼자 관리
- 키는 GitHub Secret으로 암호화
- 유출 확률 매우 낮음

---

### 2) 사용자 계정(deploy) 운영

**설정 시간:** 15~30분

**장점**
- 보안이 좋음
- 활동 추적 가능
- 실수 영향 제한적
- 팀 확대에 쉬움
- 업계 표준

**단점**
- 초기 설정 복잡함
- SSH 키 2개 관리 필요
- sudo 권한 설정 필요
- 경로 관리 복잡
- 권한 문제 발생 가능

**실제 필요성: 낮음**
- 혼자 운영 (책임 추적 불필요)
- 개인 서버 (팀 권한 관리 불필요)
- 초기 단계 (나중에 변경 가능)
- 개발 중 (보안보다 빠른 개발 중요)

---

## 시간 비교

| 작업 | Root | 사용자 계정 |
|------|------|----------|
| GitHub Secrets 설정 | 1분 | 1분 |
| 워크플로우 수정 | 1분 | 1분 |
| 서버 초기 설정 | 1분 | 5분 |
| 프로젝트 클론 | 2분 | 2분 |
| 배포 테스트 | 1분 | 1분 |
| **총 시간** | **6분** | **11분** |

---

## 나중에 변경하려면?

Root에서 시작했다가 나중에 업그레이드하는 것도 간단합니다:

```bash
# 1. 사용자 계정 생성
sudo useradd -m -s /bin/bash deploy

# 2. 프로젝트 복사
sudo cp -r /var/www/edu /home/deploy/projects/

# 3. 권한 변경
sudo chown -R deploy:deploy /home/deploy/projects

# 4. GitHub Secrets 변경 (2분)

# 5. 배포 테스트
git push origin main

완료! 5분 안에 업그레이드됨
```

## 최종 추천

### Root 선택하세요 (추천)
- 지금 배포 자동화를 빠르게 구축하고 싶다
- 개인 프로젝트다
- 복잡한 설정을 피하고 싶다
- 나중에 언제든 변경할 계획이다

### 사용자 계정 선택하세요
- 처음부터 "올바르게" 하고 싶다
- 팀이 있거나 있을 예정이다
- 이미 다른 서비스가 운영 중이다
- 15분의 초기 투자 가능하다

2. 나중에 언제든 변경 가능
3. 현재는 개발/테스트 단계
4. 프로덕션 전에 충분히 업그레이드 가능
5. 불필요한 복잡성 피하기
